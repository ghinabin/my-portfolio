# Image Guidelines for Portfolio

This document provides specifications and recommendations for all images needed in the portfolio.

## General Image Requirements

### Technical Specifications
- **Format:** JPG for photos, PNG for UI screenshots with transparency
- **Resolution:** 2x for retina displays (e.g., 2400px wide for 1200px display)
- **Optimization:** Compress images (use TinyPNG, ImageOptim, or similar)
- **File size:** Aim for under 500KB per image
- **Color space:** sRGB

### Naming Convention
```
project-name-description.jpg
```
Examples:
- `design-system-components-overview.jpg`
- `eligibility-dashboard.jpg`
- `find-care-search-interface.jpg`

## How to Add Images

### Option 1: Replace Placeholder Divs
Find divs like this:
```html
<div class="case-study-image">
    [Description]
</div>
```

Replace with:
```html
<img
    src="images/filename.jpg"
    alt="Descriptive alt text"
    class="case-study-image"
    loading="lazy"
>
```

### Option 2: Use Background Images (for decorative images)
```html
<div
    class="case-study-image"
    style="background-image: url('images/filename.jpg'); background-size: cover; background-position: center;"
    role="img"
    aria-label="Descriptive text"
>
</div>
```

## Image Optimization Workflow

1. **Capture/Export** at 2x resolution
2. **Edit** if needed (crop, adjust colors, add annotations)
3. **Optimize:**
   - Use [TinyPNG](https://tinypng.com) for lossy compression
   - Or ImageOptim for local optimization
   - Or Squoosh for fine-tuned control
4. **Add to `images/` folder**
5. **Update HTML** with proper alt text
6. **Test** on retina and non-retina displays

---

## Alt Text Guidelines

Good alt text examples:
- ✅ "Component library showing buttons, forms, and navigation elements"
- ✅ "Dashboard interface with task cards and recent activity timeline"
- ✅ "Search results page with filter sidebar showing specialty and location options"

Bad alt text examples:
- ❌ "Screenshot"
- ❌ "Image"
- ❌ "design-system.jpg"

**Rule:** Describe what someone would need to know if they couldn't see the image.
